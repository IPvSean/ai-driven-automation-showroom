= Section 7 - Red Hat Insights — CVE and Advisor Remediation
:navtitle: Red Hat Insights — CVE and Advisor Remediation

== Objective

*Estimated time: 30-40 minutes*

In this section, you will use **Red Hat Insights** to identify vulnerabilities and misconfigurations across your RHEL fleet, generate remediation playbooks, review them in a developer IDE, push them to Git, and create Job Templates in Ansible Automation Platform. This is the foundation for both human-guided and fully autonomous remediation workflows in the subsequent sections.

After completing this section, you will be able to:

* Access Red Hat Insights and navigate CVE and Advisor reports
* Generate Ansible Playbooks from Insights to remediate CVEs and Advisor recommendations
* Review AI-generated playbooks inside a developer IDE before committing
* Push validated playbooks to Git and create corresponding AAP Job Templates
* Understand the difference between CVE remediation and Advisor recommendations

[quote]
💡 **Two types of Insights remediation playbooks**: Red Hat Insights generates playbooks for **CVEs** (known vulnerabilities with a formal CVE ID) and **Advisor recommendations** (proactive best-practice recommendations from Red Hat, such as fixing SELinux policies or configurations that would not survive a reboot). Both follow the same generate → review → push → execute pattern.

== Background: Red Hat Insights

https://console.redhat.com/insights[Red Hat Insights^] is a managed service included with every RHEL subscription. It continuously analyzes your connected RHEL systems and provides:

* **Vulnerability reports** — identifies CVEs affecting your systems, with severity ratings and remediation guidance
* **Advisor recommendations** — proactive suggestions from Red Hat Support to improve stability, performance, and security (e.g., "SELinux is in permissive mode" or "system would not survive a reboot due to a bad policy")
* **Remediation playbooks** — auto-generated Ansible Playbooks that fix identified issues, ready to download and execute

[quote]
💡 **Advisor vs CVE**: Think of CVEs as *reactive* — patching known vulnerabilities. Advisor recommendations are *proactive* — fixing misconfigurations and applying best practices before they become incidents. Both are critical for a healthy RHEL fleet.

=== Review the Downloaded Playbooks

1. Open each of the four downloaded playbooks in the IDE editor.
2. Review the YAML content — verify the tasks make sense for their intended targets.
3. Pay attention to:
   * The `hosts:` target — does it match the correct RHEL instance?
   * The modules used — are they appropriate (e.g., `ansible.builtin.dnf` for patching, `ansible.posix.selinux` for SELinux)?
   * Any variables or parameters that might need adjustment for your environment

[quote]
💡 **Why manual review matters**: AI-generated playbooks are powerful but not infallible. A quick human review catches edge cases, environment-specific issues, or overly broad changes before they hit production. This is the "human in the loop" principle applied to infrastructure as code.

=== Example: CVE Remediation Playbook

Here is an example of what a CVE remediation playbook from Insights might look like:

[source,yaml]
----
- name: Remediate CVE-XXXX-XXXXX
  hosts: all
  become: true
  tasks:
    - name: Update vulnerable package
      ansible.builtin.dnf:
        name: <package-name>
        state: latest
      when: ansible_os_family == "RedHat"

    - name: Verify package version
      ansible.builtin.command:
        cmd: rpm -q <package-name>
      register: pkg_version
      changed_when: false

    - name: Display updated version
      ansible.builtin.debug:
        msg: "Package updated to {{ pkg_version.stdout }}"
----

== The General Workflow

Here is the complete flow you just completed:

[source]
----
Red Hat Insights (CVE/Advisor)
    → Generate Remediation Playbooks
        → Download to Developer IDE
            → Human Reviews Playbooks
                → Push to Git (Gitea)
                    → Sync Project in AAP
                        → Create Job Templates
----

This is the **human-in-the-loop** approach to AIOps remediation. The AI and Insights platform do the heavy lifting of identifying issues and generating fixes, but a human validates everything before it reaches production.

== Key Takeaways

* **Red Hat Insights** continuously monitors your RHEL fleet for CVEs and misconfigurations
* **Two types of remediation**: CVE patches (reactive) and Advisor recommendations (proactive)
* **IDE review** adds a human validation step between AI-generated content and production execution
* **Four playbooks** cover the breadth of remediation scenarios: vulnerability patching, advisor fixes, service configuration, and system roles
* **Git as the source of truth** ensures all playbooks are version-controlled and auditable
